# 旅行攻略制作 SOP（标准操作流程）

## 一、工作阶段

### Phase 1: 约束收集
- [ ] 确认所有已订交通（不可改的硬约束）
- [ ] 确认所有住宿（check-in/out 时间）
- [ ] 确认已订景点门票（固定时段）
- [ ] 确认航班信息（到达/出发时间）
- [ ] 标记所有"定死的"时间锚点

### Phase 2: 逐天信息研究
每天需要收集的信息清单：

#### 交通
- [ ] 景点间距离（Google Maps 步行/地铁/公交）
- [ ] 地铁线路（哪条线、哪个站、几号出口）
- [ ] 步行时间（考虑老人速度 ×1.3 系数）
- [ ] 交通费用

#### 景点
- [ ] 开放时间（当日是否为特殊日）
- [ ] 门票价格（当前最新价，注意涨价）
- [ ] 是否需要预约（提前多久？）
- [ ] 游览时长（含排队）
- [ ] 无障碍/电梯情况（老人适配）
- [ ] 寄存行李（是否需要？在哪？费用？）
- [ ] 拍照最佳位置和时间

#### 餐饮
- [ ] 每餐推荐 2-3 家具体餐厅
- [ ] 地址 + 与上一个景点的距离
- [ ] 营业时间（确认当天开门）
- [ ] 人均价格
- [ ] 推荐菜品（具体菜名）
- [ ] 是否需要预约

#### 风险
- [ ] 天气影响（室外活动的备用方案）
- [ ] 季节性关闭（桥、湖、缆车等）
- [ ] 施工/维修信息
- [ ] 安全提醒（防盗、防骗）

### Phase 3: 时间线设计
- [ ] 基于锚点向前/向后排列活动
- [ ] 每个活动标注：开始时间、持续时间、结束时间
- [ ] 活动间预留缓冲时间（最少15min）
- [ ] 午餐安排在 11:30-13:00
- [ ] 下午安排休息时间（老人需要 1-2h）
- [ ] 晚餐安排在 18:30-20:00
- [ ] 检查：总步行距离是否超过 8km？超过需调整
- [ ] 检查：连续步行时间是否超过 2h？超过需插入休息

### Phase 4: 确认与决策
需要用户确认的：
- 💰 所有花钱的决定
- 🔄 行程顺序变更
- ⚖️ 取舍选择（A还是B）
- ⏰ 具体时段选择

可以自行确定的：
- 📍 交通路线选择（最优解直接定）
- 🍽️ 餐厅推荐（给选项）
- 📸 拍照点位
- ⚠️ 注意事项
- 📝 攻略文案

### Phase 5: 攻略呈现
每天的标准模板：

```
## Day X | 日期 周X | 城市 | 主题
[一句话概括今天做什么]

### 时间线
HH:MM | 活动名称 | ⏱️时长
       📍 地址/位置
       🚇 怎么去（具体线路+站名+步行距离）
       💰 费用
       📸 拍照建议
       ⚠️ 注意事项

### 今日餐饮
🥐 早餐：地点 + 推荐
🍽️ 午餐：地点 + 推荐 + 价格
🍷 晚餐：地点 + 推荐 + 价格

### 今日费用
| 项目 | 费用 |
|------|------|
| 门票 | €XX  |
| 交通 | €XX  |
| 餐饮 | €XX  |
| 合计 | €XX  |

### 备注
- 需要提前准备的物品
- 备用方案（如天气不好）
```

## 二、质量标准

### 信息可靠性
- 价格必须来自官网（标注来源）
- 开放时间必须确认是当天（注意周一闭馆等）
- 交通时间必须基于 Google Maps 或官方时刻表
- 不确定的信息标注 ⚠️ 待确认

### 老人适配
- 每天步行总距离 ≤ 8km
- 连续步行 ≤ 2h 后必须安排休息
- 优先选择有电梯/缆车/巴士的方案
- 避免台阶过多的路线
- 下午安排 1-2h 休息时间

### 呈现原则
- 只给结论，不给讨论过程
- 精确到分钟的时间线
- 每个地点给具体地址
- 每顿饭给具体餐厅和菜名
- 用 emoji 标注类型，一目了然
- 备用方案简洁标注，不展开

## 三、Skill 设计预留

本 SOP 后续可提炼为可复用的 travel-planning skill：
- 输入：目的地、天数、旅客画像（年龄/体力/偏好）、预算、已订信息
- 处理：自动研究 → 排列时间线 → 计算距离 → 生成攻略
- 输出：结构化的每日攻略（Markdown/HTML）
- 交互：在需要决策的节点暂停，等用户确认后继续
